# 2026-W16 (Apr 13–19) — v0.3 → v0.6.5 in two days

**Headline**: The week omniscitus became a real plugin. Migrate got
teeth, uninstall went surgical, weekly summaries rolled up automatically,
plugin-update-apply closed the loop, and the Tests/Blueprint/Constellation
UI became something a non-dev can actually read.

## What happened

### Migrate productionization

See `devops/2026-04-13-migrate-productionization.md`. Anchor.yaml + footprint
tracking (#40), language prompt + CLAUDE.md integration (#37), first-class
install docs (#36), richer config template.

### Weekly summaries land

See `devops/2026-04-13-weekly-backfill-and-wrapup-rolls.md`.
`/weekly-backfill` skill (#43, #44, #45) for retroactive summaries.
`/wrap-up` auto-runs backfill on week rollover (#38). Birdview card

### Surgical uninstall + plugin-update-apply

See `devops/2026-04-14-plugin-update-apply.md`.
`/omniscitus-uninstall` reads the footprint and reverses each entry
(#48, #49, #50). `/omniscitus-update` mirrors the pattern for
upgrade-time canonical refresh (#58). SessionStart nag walks up cwd
looking for `anchor.migrate_version` and emits a second upgrade hint.

### Hook + server reliability

See `devops/2026-04-13-hooks-reliability.md`. PreCompact hook type fix
(#53) brought every hook back online. Birdview server survives the
tool-call boundary via `run_in_background: true` (#51, #52). Server
auto-increments port on `EADDRINUSE` (#54). blueprint-tracker now
project-root-scoped.

### Birdview readability pass

See `web/2026-04-14-birdview-readable-ui.md`. Side-by-side Input →
Expected with → arrow (#57). JSON syntax highlight. Schema-strategy
returns render as type pills with opt-in `describe:` / `example:`
fields (#62, #63). Blueprint + Constellation gain date + author filters
(#59). Per-page purpose hints (#64). Demo banner saga concluded.

### Demo discoverability

"View Live Demo" CTAs on the landing hero and README (#55). One-command
how-to block on the demo birdview home. Thresholds now render with
meaningful Fail/Warn/Pass bands.

## Decisions worth remembering

- **Release small and often**. Every landing PR got its own version bump
  + changelog. Made it trivial to diagnose which change caused which
  behavior change during the UI readability pass.
- **Schemas are for declaring, not comparing.** `strategy: schema`
  tests don't try to assert equality on non-serializable return types —
  they declare the type and point at the real vitest/test file.
- **Consent is non-negotiable for any external write.** Migrate,
  uninstall, and update all go through AskUserQuestion for every file
  outside `.omniscitus/`.
